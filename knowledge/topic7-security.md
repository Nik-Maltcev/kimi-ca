# Тема 7: Протоколы и безопасность

146. Протокол передачи данных — набор правил для обмена данными между системами. Примеры: HTTP (запрос-ответ, основа интернета), MQTT (издатель-подписчик, топики), TCP, SMTP, HTTPS, XMPP.

147. HTTP vs HTTPS: HTTP — данные в открытом виде, уязвимы для перехвата. HTTPS — шифрование через SSL/TLS, гарантирует конфиденциальность паролей, переписок, платежей.

148. Аутентификация — проверка «Кто ты?» (логин/пароль). Авторизация — «Что тебе разрешено?» (права доступа). JWT (JSON Web Token): Header (шифрование) + Payload (данные пользователя и роли) + Signature (цифровая подпись). Сервер не хранит токен — клиент отправляет в заголовке Authorization: Bearer <token>, сервер проверяет подпись.

149. При переходе по URL: парсинг URL (протокол, домен, путь, параметры) → DNS Lookup (домен → IP) → TCP 3-way handshake → HTTP запрос → сервер обрабатывает (обращается к БД) → HTTP ответ → браузер рендерит (HTML→DOM, CSS→CSSOM, JavaScript, загрузка ресурсов).

150. Шифрование — математическое преобразование данных в нечитаемый вид. Расшифровка только с ключом. Асимметричное (публичный + приватный ключ) — безопаснее, но медленнее. Симметричное — быстрее. Применение: HTTPS, VPN, BitLocker, WhatsApp, SSL/TLS, цифровые подписи.
