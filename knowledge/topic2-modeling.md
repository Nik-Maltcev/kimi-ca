# Тема 2: Нотации моделирования

23. Нотации моделирования: ER-диаграммы (нотации Чена и Мартина), IDEF (IDEF0 для функций, IDEF1 для инфопотоков, IDEF3 для процессов), EPC для событийных процессов, C4 (от контекста до кода).

24. Элементы BPMN: События (начало, промежуток, конец), Задачи (прямоугольники — конкретный шаг работы), Шлюзы (ромбы — разветвление/слияние логики), Потоки управления (стрелки — порядок действий). Дополнительно: Дорожки (swimlanes — исполнители), Артефакты (объекты данных, документы, комментарии), Подпроцессы (вложенные процессы).

25. Типы шлюзов в BPMN: XOR (исключающее ИЛИ — строго один путь), AND (параллельное И — все пути одновременно), OR (неисключающее ИЛИ — один или несколько путей), событийные шлюзы (XOR и AND по событиям — поток идет туда, где быстрее наступит событие), комплексные шлюзы.

26. События в BPMN: Стартовые (тонкая линия круга), Промежуточные (двойная линия), Конечные (жирная линия). Значки триггеров: таймер, сообщение, ошибка, эскалация.

27. Виды диаграмм UML: Use case (роли и функции), Activity (логика процессов, аналог BPMN), Sequence (обмен вызовами во времени), Class (структура кода), State Machine (жизненный цикл и статусы объекта).

28. Фреймы в Sequence диаграмме: loop (цикл — шаг выполняется несколько раз), alt (if/else — альтернативные ветки), opt (опциональный фрагмент — выполняется при истинности условия), par (параллельное выполнение), ref (ссылка на стороннюю диаграмму).

29. Class Diagram vs ER Diagram: Class Diagram — для кода (сущности + методы + инкапсуляция). ER Diagram — только для проектирования БД (сущности, атрибуты, ключи, связи, без методов).

30. Use case диаграмма: Актор (человечек) соединяется Ассоциациями с Прецедентами (овалы с функциями). Связи между овалами: include (обязательное включение), extend (необязательное расширение).
